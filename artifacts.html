This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems complying with the implementation guide are expected to declare conformance to one or more of the following capability statements.

PACIO Cognitive Status Capability Statement This Capability Statement defines the expected capabilities of a PACIO Cognitive Status FHIR server conforming to the PACIO Cognitive Status Implementation Guide.

Structures: Resource Profiles

These define constraints on FHIR resources that need to be complied with by conformant implementations

PACIO Bundled Cognitive Status An point in time bundle of cognitive status observations for a patient.
PACIO Cognition Questionnaire Response A completed, or sections of a completed, CMS PAC assessment.
PACIO Cognitive Status An exchange of cognitive status data for a patient.
Use of Device A device used by a Patient during a cognitive status assessment.
